What a Transformer really does (in one pass)

A Transformer maps an input sequence of token IDs to a probability distribution for the next token. The key mechanism is self-attention: every token can “look at” other tokens to decide what information is relevant.

Big picture:
  • Embeddings turn tokens into vectors.
  • Attention mixes information across positions (with masking for decoding).
  • Residual + LayerNorm keep training stable and preserve gradients.
  • FFN (feed-forward network) adds nonlinearity per token.
  • Projection to logits gives scores for every vocab token.
  • Sampling / decoding chooses the next token.

1) Embeddings deeper dive

An embedding is a learned lookup table: for each token ID, you retrieve a vector. But embeddings aren’t just IDs→vectors; they’re the model’s learned coordinate system.

Why vectors at all?

  • Similarity matters: attention scores are computed from dot-products of these vectors.
  • Arithmetic emerges: relationships (e.g., “king - man + woman”) can become approximately linear in embedding space.
  • Generalization: unseen combinations can still land near familiar regions.

Token identity vs context

The embedding alone encodes token identity. Context is added when the model applies attention and feed-forward layers. In other words: embeddings are necessary, but not sufficient.

2) Positional information: more than “sequence order”

Attention is permutation-invariant by default, so the model needs a way to know where tokens occur. Positional encodings add position-dependent signals.

Common variants

  • Sinusoidal encodings: fixed functions of position; enable extrapolation.
  • Learned absolute embeddings: each position has a learned vector.
  • Relative encodings: attention depends on relative offsets (often improves long-range behavior).
  • RoPE (rotary): rotates query/key vectors based on position.
Watch out: if positional encoding doesn’t match training/test lengths or distribution, behavior can degrade.

3) Attention masks & decoding modes

During training with teacher forcing, you often see full-sequence visibility. During autoregressive decoding, you must prevent the model from “cheating” by attending to future tokens.

Mask types you’ll hear about

  • Causal mask: token t can attend only to positions ≤ t.
  • Padding mask: ignore padded tokens in variable-length batches.
  • Cross-attention mask: in encoder-decoder models, control which source positions are visible.

Intuition

Masking changes the attention pattern, which changes the information each position can carry. Many “it doesn’t work” issues are actually mask or decoding bugs.

4) Multi-head attention: different “views” per head

Multi-head attention runs several attention mechanisms in parallel (heads). Each head learns a different similarity metric.

What do heads learn?

  • Local heads may focus on nearby tokens.
  • Copy heads may track lexical overlap or repeated patterns.
  • Syntax heads may align with grammatical structures.
  • Coreference heads may connect pronouns to antecedents.

Per-head intuition (not one-size-fits-all)

It’s tempting to think all heads do the same thing with different parameters. In practice, they specialize—so removing heads can degrade different behaviors differently.

5) Residual connections & LayerNorm: pre vs post

Residual pathways help gradients flow; LayerNorm stabilizes activations. Two common layouts: Pre-LN (normalize before sublayer) and Post-LN (after).

Pre-LN intuition

  • Often trains more stably for deep Transformers.
  • Makes optimization easier because residual branch starts closer to identity.

Post-LN intuition

  • Normalize after sublayer; can work well but may be more sensitive to depth / learning rate.
Why it matters: subtle architectural choices can change convergence, calibration, and sometimes long-range behaviors.

6) FFN blocks: token-wise transformation with nonlinearity

The feed-forward network (FFN) is applied to each position independently. It expands dimensionality, applies a nonlinearity, then projects back.

While attention mixes information across positions, the FFN refines what each position “means” after mixing. Think of attention as communication and FFN as interpretation.

7) Decoding strategies & the generation loop

Autoregressive generation repeatedly: (1) run the model, (2) pick the next token, (3) append it, and continue.

Sampling vs search

  • Greedy: pick argmax. Coherent but can get stuck in bland patterns.
  • Temperature: smooths or sharpens probability distribution.
  • Top-k: restrict to k most likely tokens.
  • Top-p (nucleus): restrict to smallest set whose cumulative prob ≥ p.
  • Beam search: keeps multiple hypotheses; can improve some tasks but may hurt diversity.
Common confusion: temperature affects distribution shape, while top-k/top-p changes support. They’re different knobs.

8) Training objective: teacher forcing & next-token prediction

Transformers are commonly trained with next-token prediction. Given a sequence, you shift it by one token and ask the model to predict the “future” token at each position.

Teacher forcing

During training, the model is given the ground-truth previous tokens (not its own generated tokens). This reduces error compounding and makes optimization feasible.

Why this objective works

  • It turns learning into a standard supervised classification problem over vocab tokens.
  • Gradients flow through the entire sequence in parallel (unlike step-by-step sampling).
  • It naturally supports autoregressive generation at inference time.
Practical note: evaluation metrics like perplexity relate to cross-entropy loss under the same next-token objective.

9) Common failure modes (and what they reveal)

  • Repetition loops: can indicate overly peaked distributions or decoding settings that lack diversity.
  • Hallucination: the model predicts plausible continuations rather than verified facts.
  • Attention confusion: wrong masks or positional encoding mismatches can cause incoherence.
  • Degenerate sampling: low temperature + no support restriction can collapse into repetitive tokens.
  • Long-context drift: some positional schemes degrade outside training length.

10) End-to-end walkthrough: tying it back to the simulator

Let’s connect the story to the interactive prompt. Suppose your prompt text becomes token IDs: [t₁, t₂, …, tₙ]. A real Transformer doesn’t “pick a token” from a simple if/then rule. Instead, it builds representations for each position, runs attention and FFN layers, then produces logits for the next position.

Still, you can map the simulator knobs to the mental model of decoding:

  • Attention emphasis (in the diagram) corresponds to how strongly the next-token prediction depends on previous tokens.
  • Residual + LayerNorm corresponds to stabilizing the representation: the model tends not to wildly overwrite everything each layer.
  • Logits projection corresponds to the final scores for each vocabulary entry.

The simulator then uses temperature and top-k to sample from the final distribution. In other words: the diagram highlights stages of computation, while temperature/top-k control how you explore the model’s likely next tokens.

Walkthrough exercise:
  1. Use a short prompt: “In a distant future, the city”.
  2. Generate once with Temperature ≈ 0.4 and Top-k ≈ 10.
  3. Increase temperature to ≈ 1.2 while keeping top-k fixed. Observe whether more varied continuations appear.
  4. Decrease top-k to ≈ 5 while keeping temperature moderate. Observe how the simulator becomes conservative.
  5. Repeat with a different prompt and compare how the “shape” of candidate tokens shifts.

Deep dive: attention in 60 seconds

What happens inside one attention head?

  • Project inputs to queries (Q), keys (K), and values (V).
  • Score each token pair with a similarity (often dot-product).
  • Mask future positions in causal decoding.
  • Normalize scores with softmax → attention weights.
  • Mix values using weighted sum.
Try this: when decoding, attention weights near the end usually matter most.

Concept checklist

  • Logits are unnormalized scores.
  • Softmax converts logits to probabilities.
  • Embeddings define the model’s geometry.
  • Positional encodings inject order information.
  • Residual connections preserve information.
  • LayerNorm stabilizes training.

Common architectural variants

Encoder-only

BERT-style masked language modeling; no autoregressive mask for generation.

Decoder-only

GPT-style causal self-attention for next-token generation.

Encoder-decoder

Transformer for seq2seq; cross-attention links target to source tokens.

Intuition: why residuals help

Residual connections mean each layer can learn “deltas” rather than replace everything. That keeps representations from drifting too far while learning new refinements.

Another way to say it: early layers can preserve the identity path; later layers add semantics.
